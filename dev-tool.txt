# Guide to Using Relative Paths in GitHub Pages

## Understanding Relative Paths

- `./` refers to the current directory
- `../` refers to the parent directory
- You can combine them: `../../` goes up two directories

## How to Use Relative Paths in GitHub Pages

### For Project Sites (`username.github.io/project-name/`)

Remember that your site is served from the root of your repository but under the `/project-name/` path.
